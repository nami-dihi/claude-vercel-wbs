# /commit

변경된 파일을 확인하고 지정된 템플릿에 맞춰 커밋을 진행하는 스킬입니다.

---

## 절차

### 1. 변경된 파일 상태 확인

현재 저장소의 상태를 파악하여 stage에 올라간 항목과 올라가지 않은 항목을 확인합니다.

```bash
git status
```

### 2. 사용자에게 커밋 대상 확인

가져온 상태를 바탕으로 Stage에 올라간 항목(Staged)과 올라가지 않은 항목(Unstaged/Untracked)을 명확하게 리스트업하여 사용자에게 보여줍니다.

> "현재 변경된 파일은 다음과 같습니다.
> 
> **✅ Staged 항목:**
> - `src/example1.ts`
> 
> **⚠️ Unstaged/Untracked 항목:**
> - `src/example2.ts`
> 
> 어떤 파일들을 커밋할까요? (예: 전부 커밋해줘, Staged 항목만 커밋해줘)"

### 3. 파일 Staging (필요시)

사용자의 지시에 따라 추가로 커밋할 파일이 있다면 `git add` 명령어를 통해 stage에 올립니다.

### 4. 커밋 메시지 작성 및 확인

작업 내역을 바탕으로 커밋 메시지를 초안으로 작성하여 사용자에게 제안하거나, 사용자로부터 내용을 전달받습니다.
**커밋 메시지는 반드시 `.agents/rules/commit-template.md`의 형식을 엄격하게 따릅니다.**

- `<type>: #<이슈번호> <이슈 제목 요약>` 형태의 제목
- 변경 내용에 대한 bullet(`-`) 리스트
- 마지막 줄에 `Co-Authored-By` 포함

### 5. 커밋 실행

사용자가 커밋 메시지를 승인하면 커밋을 실행합니다.

```bash
git commit -m "..."
```

---

## 규칙

- 커밋을 실행하기 **전에** 반드시 대상 파일을 리스트업하고 사용자의 승인을 받아야 합니다.
- 커밋 메시지는 `.agents/rules/commit-template.md` 템플릿 구조를 절대적으로 준수해야 합니다.
- 한국어로 대화합니다.
