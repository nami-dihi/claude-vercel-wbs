# /commit-staged

현재 Stage에 올라간 파일들의 변경사항을 분석하여 지정된 템플릿에 맞춰 커밋을 진행하는 스킬입니다. 
`/commit`과 달리 처음에 어떤 파일을 커밋할지 묻는 단계를 생략하고, 이미 Stage된 항목에 대해서만 바로 커밋 메시지 초안 작성을 시작합니다.

---

## 절차

### 1. Staged 상태 확인 및 분석

현재 저장소 상태를 파악하여 Stage에 올라간 변경사항의 내용을 확인합니다.

```bash
git status
git diff --cached
```

- 만약 Stage에 올라간 파일이 없다면, 사용자에게 "Stage에 올라간 파일이 없습니다. 파일을 add 한 후 다시 실행해 주세요."라고 알리고 스킬을 종료합니다.

### 2. 커밋 메시지 작성 및 확인

분석한 작업 내역을 바탕으로 커밋 메시지를 초안으로 작성하여 사용자에게 제안합니다.
**커밋 메시지는 반드시 `.agents/rules/commit-template.md`의 형식을 엄격하게 따릅니다.**

- `<type>: #<이슈번호> <이슈 제목 요약>` 형태의 제목
- 변경 내용에 대한 bullet(`-`) 리스트
- 마지막 줄에 `Co-Authored-By` 포함

> "현재 Stage에 올라간 파일들을 바탕으로 다음 커밋 메시지를 작성했습니다. 이대로 커밋을 진행할까요? 수정할 부분이 있다면 알려주세요."
>
> (커밋 메시지 초안 출력)

### 3. 커밋 실행

사용자가 커밋 메시지를 승인하면 커밋을 실행합니다.

```bash
git commit -m "..."
```

---

## 규칙

- 커밋을 실행하기 **전에** 반드시 커밋 메시지 초안을 보여주고 사용자의 승인을 받아야 합니다.
- 커밋 메시지는 `.agents/rules/commit-template.md` 템플릿 구조를 절대적으로 준수해야 합니다.
- 한국어로 대화합니다.
